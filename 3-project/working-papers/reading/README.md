# Reading

Primary-source research log for the doctoral programme. A living record of what exists to be read, what's been read, and what each paper contributed to the investigation.

## Contents

**[reading-log.md](reading-log.md)**: the bibliography. Organised by author with priority markers (P1 = minimum to write credibly, P2 = deepens engagement, P3 = reference), reading status, and access notes. Updated as sources are identified, accessed and read.

**[notes/](notes/)**: per-paper reading notes, one file per source. See `notes/README.md` for the convention. When a paper moves from `reading` to `read` status in the log, a note file in this directory should exist.

## Convention

The reading log is a *living document*, not a static list. Update it whenever:

- A new source is identified as worth reading
- Status changes on any entry (`not started` → `reading` → `read` → `reread`)
- Access notes change (PDF located, library access established, book ordered)
- Priority shifts based on what primary sources reveal

When a paper is finished, three things should happen in this order:

1. A note file is written in `notes/`
2. The log entry gets a link to that note
3. The log entry's status updates

This keeps the log in sync with the substance of what's been extracted.

## What belongs here

- Bibliography entries and access tracking (`reading-log.md`)
- Per-paper engagement notes (`notes/`)

## What doesn't belong here

- Synthesis documents that pull across multiple sources (those go in `../writing/` or back into the relevant theory docs)
- Drafts of proposal revisions (those go in `../proposals/`)
- Ad-hoc reactions that aren't tied to a specific source (those go in `../writing/`)
